Brain4Cars: Learning to Anticipate Driving Maneuvers

Under the guidance of Prof. Ashutosh Saxena, Robot Learning Lab, Computer Science, Cornell University (May 2015 - July 2015)


Brain4Cars takes input from driver-facing and road-facing cameras, and predicts what maneuvers the driver is going to perform in the next few seconds. My contribution to the project was in the following areas:

  • Incorporated a Constrained Local Neural Field model for improved head pose tracking of the driver.
  • Implemented Gaussian Mixture Model-based initialization, and LBFG-S optimization for training Autoregressive Input Output Hidden Markov Models
  • The above changes improved the performance of the anticipation system from a Precision/Recall of 77.4/71.2 to 86.7/78.2.

Visual Odometry

Under the guidance of Prof. KS Venkatesh, Computer Vision Lab, Electrical Engineering, IIT-Kanpur (Aug 2014 to Dec 2014)

Stereo Visual Odometry

  • Jiang2014: Model Based ICP: 3D points triangulated from stereo data, inliers detected via the use Iterative Closest Point Algorithm that uses a 1-DOF motion model for initial estimate. Efficient PnP Algorithm is then used on the selected inliers to obtain the final rotation and translation
  • Howard2008: Inlier detection using an assumption of scene rigidity. Problem reduced to finding the maximum clique in a graph, solved using a heuristic. Levenberg-Marquardt used for minimizing the reprojection error on the selected inliers. Implmentation available on Github.

Range Flow (RGBD version of Optical Flow)

Under the guidance of Prof. KS Venkatesh, Computer Vision Lab, Electrical Engineering, IIT-Kanpur (May 2014 - July 2014)

  • Formulated and implemented two approaches for estimation of Range Flow from RGB-D data, one an extension of Lucas-Kanade to 3D, and the other an extension of Horn-Schunck.
  • A third approach attempting to capture the actual flow (instead of apparent flow) was also implemented.
  • The third RGBD Flow approach was evaluated against data generated using a highly accurate 6-DOF tracking device, the PST-110.
  • Captured RGBD Data from a Microsoft Kinect using OpenNI and OpenCV libraries, and did all the processing using C++ in OpenCV.

Landmark Based Robot Localisation using a Kinect Sensor

Under the guidance of Arjun Bhasin, Project Engineer at Mechatronics Lab, Mechanical Engineering, IIT-Kanpur (Dec 2013)

  • Geometric Triangulation was used to determine the pose (modelled as a set of random variables with Gaussian Distribution) of a robot, from information obtained from noisy landmarks.
  • A Microsoft Kinect was used to identify the landmarks (using color histogram based models), CAMshift algorithm was used to track these landmarks, and bearing measurements were calculated using the depth data.
  • An error model for the data obtained from a Kinect was used along with the Error Propagation Law to arrive at the uncertainty in the final pose computed using the Geometric Localisation Algorithm.
